What cookies does the toolbar set

Created: 2026-06-19T11:47:40.486299+00:00
Question: What cookies does the toolbar set?

Answer

Non-technical explanation

The toolbar stores two main browser cookies: one to remember whether the toolbar should stay enabled across page loads, and one to store the user’s selected accessibility preferences.

Technical explanation

The key cookie names used by toolbar code are Recite.Persist and Recite.Preferences. Recite.Persist is written/read by Recite.js and used by launcher logic to auto-launch when set to true. Recite.Preferences is saved by the preferences module. Storage implementation is in Recite/Storage/Cookie.js (cookie writer/reader with domain/secure/samesite options). The launcher also checks localStorage key recite.persist, which is not a cookie.

Recommended next steps

  1. Treat Recite.Persist as toolbar on/off persistence state.
  2. Treat Recite.Preferences as user preference payload.
  3. If auditing storage, check both cookies and localStorage key recite.persist.
  4. Confirm exact lifetime/domain flags from cookieLife, Storage.Cookie.cookiesDomain, and Storage.Cookie.secure settings.

What to verify

  • Runtime cookie values in browser devtools for your environment.
  • Whether integration uses cookie or localStorage preference engine settings.
